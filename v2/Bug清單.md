# v2 Bug 清單

> 這份只整理目前屬於 bug、狀態錯誤、路徑錯誤、流程錯亂、UI 操作問題的項目。
> 不包含新增功能或後續擴充需求。

## 一、狀態與恢復

- ［Bug］程式重開後固定回到 Step 1，沒有恢復到上一輪未完成的位置。
- ［Bug］`current_step` 只有欄位，實際沒有被保存與恢復。
- ［Bug］程式不會根據專案資料夾現況判斷目前進度，導致實際做到一半時仍被當成新一輪。
- ［Bug］若上一輪中斷，系統不會標記為未完成，容易直接跳成下一輪或重置流程。
- ［Bug］切換專案後，工作流程狀態沒有切到該專案自己的進度。

## 二、輪次與文件判斷

- ［Bug］本輪文件使用固定名稱（如 `SPEC_本輪.md`），容易和上一輪或修正版混淆。
- ［Bug］舊文件與新文件並存時，系統沒有明確指定哪一份才是本輪唯一依據。
- ［Bug］上一輪文件沒有自動歸檔，導致本輪與舊輪資料混在一起。
- ［Bug］新一輪開始時，舊的共識、操作清單、變更記錄沒有被清楚標示為舊資料。
- ［Bug］每輪備份雖然有做，但只是整包備份，缺乏明確輪次對應，回查困難。

## 三、AI 讀錯檔 / 提示詞錯誤

- ［Bug］系統沒有直接把「現在是第幾輪」寫進關鍵提示詞，AI 容易靠猜測或去讀錯文件。
- ［Bug］提示詞使用模糊固定檔名，讓審查員與執行者容易讀到舊檔。
- ［Bug］審查員提示詞邊界不夠硬，AI 可能擴大成審查整個專案。
- ［Bug］執行者提示詞沒有足夠限制在本輪共識與本輪規格，可能被舊資料污染。
- ［Bug］當指定文件不存在或路徑失效時，AI 會自行掃描整個專案，進一步讀到錯的舊文件。
- ［Bug］缺少明確的 fallback 規則，導致 AI 找不到檔案時亂找替代檔。
- ［Bug］勾選圖表 / CSS 視覺修改後，樣式規格書沒有被穩定地列為必讀文件。

## 四、路徑與檔案定位

- ［Bug］專案名稱、父路徑、完整路徑語意混亂，容易組出錯誤路徑。
- ［Bug］CLI 啟動指令的路徑組合可能錯誤，且斜線格式不一致。
- ［Bug］修改專案路徑或代碼資料夾後，相關顯示與指令沒有立即刷新。
- ［Bug］相對路徑策略過度依賴固定工作目錄，工作目錄一變就可能找不到檔案。
- ［Bug］開場指令中的文件引用在不同啟動位置下不一定能正確定位。

## 五、切換專案同步失效

- ［Bug］切換專案後，初始設定欄位沒有完整更新。
- ［Bug］切換專案後，CLI 指令沒有完整更新。
- ［Bug］切換專案後，角色開場指令內容沒有完整更新。
- ［Bug］切換專案後，輪次顯示沒有完整更新。

## 六、工作流程邏輯錯亂

- ［Bug］審查通過後，系統沒有自動更新共識文件，只能依賴使用者手動補流程。
- ［Bug］專案狀態文件目前只更新輪次與時間，沒有同步整理已完成項、未完成項、本輪任務。
- ［Bug］快速模式沒有被完整納入輪次狀態管理，走完後容易缺少一致的流程紀錄。

## 七、UI / 操作問題

- ［Bug］初始頁按鈕操作後，畫面會上下跳動。
- ［Bug］啟動窗口內容過長時，複製按鈕與模型建議被擠出主要可視區域。
- ［Bug］工作流程 Step 2 之後，按鈕可能跑出畫面，需要手動滾很久才看得到。
- ［Bug］整體滾動行為不順，使用者常常必須先捲完整段內容才能操作下一步。
- ［Bug］CLI 指令選擇彈窗尺寸不穩，確認按鈕可能落在畫面外。

## 八、可編輯性與即時刷新

- ［Bug］工作流程中生成的提示內容多為唯讀，無法直接微調後再複製。
- ［Bug］角色切換 CLI / 模型後，開場指令不會即時刷新。
- ［Bug］切換到 Codex 類型時，對應的提示詞與記憶檔切換不夠即時或不夠穩定。
